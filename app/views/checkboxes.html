Checkboxes

Checkboxes allow the user to select one or more options from a set of values.

Usage Guidelines

When to use

  • Checkboxes can be used when the user is able to select more than one option.
  • Checkboxes can be used when the user needs to select that they acknowledge something.
  • Checkboxes allow all available options to be visible to the user.

When not to use

  • Do not use if the user can select only one option. Use a radio button or a toggle instead.

Interaction

Users should be able to select either the checkbox or the text to make their selection. Consider the order in which options appear. In most cases alphabetical order will work best. Keep text short and ensure the user can clearly associate the label with the input. Be careful if you set a default value — this can discourage the user from making a selection or unconsciously influence their choice. Groups of checkbox options can appear in two formats: vertically-oriented or inline.

Validation

Checkboxes should be validated only after the user has made a selection or submited their selection. Checkboxes should never use the Success validation state because users are always picking from pre-validated list of options. Read the Validation & Errors section for more information about how and when to use validation.
